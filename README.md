# Team Workboard

一个适合局域网团队内部使用的轻量事项透明板。

它主要解决这些问题：

- 团队成员现在各自在做什么
- 哪些事项今天要收口
- 哪些事项是多人协作
- 完成后怎么归档和回看

这不是重型项目管理平台，而是一个更容易被持续更新的团队看板。

## 当前能力

- 无登录版本，使用“成员选择”标记本次更新人
- 今日总览、成员看板、成员管理、事项详情、归档检索
- 新建事项时统一记录标题、负责人、事项类型、目标时间、完成标准、GitHub 链接
- 支持个人事项和多人协作事项
- 支持一句话更新、状态流转、完成归档
- 目标时间使用结构化时间存储，并统一显示为“今天 18:00 / 明天 12:00”这类口径

## 最近更新

- 成员删除改为软删除，只会被“未完成事项关联”阻止；已完成事项不会再拦住删除
- 事项详情页改成上下结构：上方先选当前事项，下方再看该事项详情和时间线
- 当前事项列表改成卡片式展示，当前选中项会高亮
- 事项详情支持直接编辑标题、负责人、时间、协作成员、进展、下一步等信息
- 事项详情支持“整体完成关闭”，不用再从成员卡片逐个收尾
- 常用动作改成明确的按钮组，减少误解

## 使用约束

- 只有事项参与人员可以更新、编辑或完成该事项
- 事项参与人员指：主负责人和协作成员
- 前端会自动收窄“更新人”可选范围，后端也会做强制校验
- 这个版本适合高信任团队透明协作，不适合强审计、强权限隔离场景

## 技术栈

- Node.js 内置 HTTP 服务
- 原生 HTML / CSS / JavaScript 单页前端
- JSON 文件持久化

## 本地运行

```bash
npm run dev
```

默认地址：

```text
http://localhost:4173
```

服务启动后会额外打印一个局域网可访问地址，方便同网段团队成员打开。

## 数据说明

- 运行时数据默认写入 `data/store.json`
- 这个文件已加入 `.gitignore`，避免把真实团队数据提交进仓库
- 如果本地没有 `data/store.json`，服务启动时会自动生成一份示例数据
